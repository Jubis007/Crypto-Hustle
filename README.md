# Crypto Hustle Lite 🪙📈

If fighting with hidden `.env` files and getting rate-limited by APIs was an Olympic sport, I’d be bringing home the gold this week. 🥇😅 

I just wrapped up my newest React project: **Crypto Hustle Lite!** Built during my Intermediate Web Development course with CodePath, this application is a live cryptocurrency dashboard that fetches, filters, and displays real-time market data and news. Moving from static data to massive, live API endpoints was a huge step up, and it forced me to really lock down my understanding of component lifecycles.

A few technical highlights from this build:

* 🔄 **Complex State Management:** Utilized `useState` and `useEffect` to manage and render massive data arrays asynchronously.
* 🔍 **Dynamic Filtering:** Engineered a custom search algorithm that instantly filters through nested JSON objects to display matching cryptocurrencies based on user keystrokes.
* 🧩 **Modular Architecture:** Broke the UI down into reusable, self-contained components (like `CoinInfo`) that independently fetch and manage their own specific data streams.
* 🛡️ **Bulletproof Error Handling:** Implemented strict data-type checking and conditional rendering to prevent fatal app crashes when APIs inevitably return incomplete data or trigger rate limits.

It was an awesome experience learning how to gracefully handle asynchronous data and edge cases. On to the next build! 🚀
